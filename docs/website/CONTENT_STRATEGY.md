# DhanRakshak Website - Content Strategy

## Content Philosophy

The DhanRakshak website content follows the same principles as the app:

- **Calm and trustworthy** - No hyperbolic marketing speak
- **Clear and concise** - Respect the reader's time
- **Privacy-focused** - Emphasize data control and transparency
- **Benefit-driven** - Focus on "why" not just "what"
- **Accessible** - Written for clarity, not complexity

## Writing Guidelines

### General Principles

1. **Lead with benefits, not features**
   - ‚úÖ "Understand your spending patterns with beautiful visualizations"
   - ‚ùå "Our app has charts and graphs"

2. **Use active voice**
   - ‚úÖ "Track expenses effortlessly"
   - ‚ùå "Expenses can be tracked"

3. **Be specific, not vague**
   - ‚úÖ "Your data is encrypted and stored locally on your device"
   - ‚ùå "We care about your security"

4. **Avoid marketing fluff**
   - ‚úÖ "Local-first privacy"
   - ‚ùå "Revolutionary next-gen paradigm-shifting privacy"

5. **Use short sentences and paragraphs**
   - Aim for 15-20 words per sentence
   - 2-3 sentences per paragraph
   - Use bullet points for lists

### Word Choice

**Prefer:**
- "you" and "your" (second person)
- Simple words over jargon
- "helps" over "enables"
- "shows" over "visualizes"
- "free" over "gratis"

**Avoid:**
- "leverage", "utilize", "synergy"
- Exclamation marks (use sparingly)
- ALL CAPS (except acronyms)
- Marketing buzzwords

### Formatting

- **Headings:** Sentence case, not title case
  - ‚úÖ "Why DhanRakshak?"
  - ‚ùå "Why DhanRakshak Is The Best Personal Finance App"

- **Buttons:** Action verbs, clear outcomes
  - ‚úÖ "Download for Android"
  - ‚ùå "Click Here"

- **Links:** Descriptive, not generic
  - ‚úÖ "Read our Privacy Policy"
  - ‚ùå "Click here for more info"

## Page-by-Page Content Guidelines

### Home Page

**Purpose:** Introduce the app, communicate value, drive downloads

**Sections:**
1. **Hero**
   - Tagline: "Personal finance, made simple and private."
   - One-line description of the app
   - Primary CTA: "Download for Android"
   - Secondary CTA: "Learn More"

2. **Screenshot Showcase**
   - Title: "Beautiful, Intuitive Design"
   - Brief description of UI philosophy
   - 4 key screenshots with captions

3. **Value Propositions**
   - Title: "Why DhanRakshak?"
   - 5 cards with icon, title, and 1-2 sentence description
   - Focus on benefits, not features

4. **Trust Section**
   - Title: "Built on Trust and Transparency"
   - 2-3 paragraphs on privacy philosophy
   - Link to Privacy Policy

5. **CTA Banner**
   - Title: "Ready to Take Control of Your Finances?"
   - Download buttons + QR code
   - Link to GitHub

**Tone:** Welcoming, benefit-focused, reassuring

### Features Page

**Purpose:** Detailed feature explanations for interested users

**Sections:**
1. **Hero**
   - Title: "Everything You Need to Manage Your Finances"
   - Brief overview paragraph

2. **Feature Blocks** (8 features)
   - Each feature: Icon, title, description, optional screenshot
   - Descriptions: 2-3 sentences, benefit-focused
   - Alternating layout (left/right)

3. **CTA Banner**
   - Same as home page

**Tone:** Informative, detailed, helpful

**Feature Descriptions:**
- Focus on user benefits
- Include specific use cases
- Mention privacy implications where relevant

### Privacy Policy

**Purpose:** Legal requirement, trust building

**Sections:**
1. Data Storage (what's local)
2. Cloud Sync (optional, how it works)
3. Splitwise Integration (what we access)
4. Analytics & Tracking (we don't)
5. Permissions (why we need them)
6. Your Control (what you can do)
7. Contact (how to reach us)

**Tone:** Professional, clear, transparent

**Guidelines:**
- Use simple language, not legalese
- Explain "why" behind each practice
- Be honest about limitations
- Make it easy to contact us

### Terms of Use

**Purpose:** Legal protection, set expectations

**Sections:**
1. Purpose of the App
2. User Responsibility
3. Third-Party Integrations
4. Availability and Updates
5. Limitation of Liability
6. No Warranty
7. Acceptable Use
8. Contact

**Tone:** Professional, fair, clear

**Guidelines:**
- Balance legal protection with readability
- Don't hide terms in jargon
- Be clear about user responsibilities
- Include dispute resolution

### Contact Page

**Purpose:** Provide support channels

**Sections:**
1. Email Support (general inquiries)
2. GitHub Issues (bug reports, features)
3. Before You Contact (helpful resources)

**Tone:** Helpful, approachable

**Guidelines:**
- Set expectations for response time (if applicable)
- Direct users to appropriate channel
- Link to FAQs/docs before asking

## SEO Content Guidelines

### Meta Titles

**Format:** `[Page Name] - DhanRakshak`

**Examples:**
- "DhanRakshak - Personal finance, made simple and private"
- "Features - DhanRakshak"
- "Privacy Policy - DhanRakshak"

**Length:** 50-60 characters

### Meta Descriptions

**Guidelines:**
- 150-160 characters
- Include primary keyword
- Focus on value proposition
- Include call-to-action

**Examples:**
- Home: "A privacy-first, local-first personal finance tracking app for Android. Track expenses, manage groups, set savings goals, and sync with Splitwise."
- Features: "Explore all the powerful features of DhanRakshak: expense tracking, group management, Splitwise sync, analytics, savings goals, and more."

### Headings (H1, H2, H3)

- **H1:** One per page, primary topic
- **H2:** Section headings, keyword-rich
- **H3:** Subsections, support H2 topics

### Keywords

**Primary:**
- Personal finance app
- Expense tracking app
- Privacy-first finance
- Local-first finance
- Splitwise alternative

**Secondary:**
- Budget app Android
- Savings goal tracker
- Financial analytics
- Group expense management
- Recurring expense tracker

### Internal Linking

- Link to Privacy Policy from features mentioning privacy
- Link to Features from value props on home
- Link between related features
- Use descriptive anchor text

## Content Maintenance

### Update Frequency

- **Home/Features:** Update when major features added
- **Privacy/Terms:** Update when data practices change
- **Contact:** Update if support channels change
- **Screenshots:** Update when UI changes significantly

### Version Control

- Date all Privacy Policy and Terms updates
- Keep changelog of major content changes
- Note version in footer

### Content Review Checklist

Before publishing any content updates:

- [ ] Spelling and grammar check
- [ ] Links work (internal and external)
- [ ] Tone matches brand voice
- [ ] No marketing fluff or hyperbole
- [ ] Privacy implications clearly stated
- [ ] SEO meta tags updated
- [ ] Mobile-friendly formatting
- [ ] Accessible (headings, alt text, etc.)

## Restricted Content

**Do NOT include:**
- Exaggerated claims ("best", "perfect", "revolutionary")
- Competitive comparisons without evidence
- User testimonials (unless verified)
- Download counts or ratings (unless verifiable)
- Promises of financial advice
- Guarantees of security (use "designed for security" instead)

## Call-to-Action (CTA) Strategy

### Primary CTAs

**Home page:**
- "Download for Android" (hero, footer banner)
- "Learn More" (secondary, links to features)

**Features page:**
- "Download for Android" (footer banner)

**Contact page:**
- Email link, GitHub Issues link

### CTA Best Practices

1. Use action verbs
2. Be specific about outcome
3. Make CTAs stand out visually
4. Place above the fold (hero)
5. Repeat at natural exit points (end of page)

## Accessibility Content Guidelines

### Alt Text for Images

**Screenshots:**
```html
alt="DhanRakshak Home Screen showing monthly expenses and budget progress"
```

**Icons:**
```html
alt="Privacy shield icon"
```

**Decorative images:**
```html
alt="" (empty, so screen readers skip)
```

### Link Text

- Avoid "click here" or "read more"
- Use descriptive text: "Read our Privacy Policy"
- If link opens in new tab, indicate: "View on GitHub (opens in new tab)"

### Headings

- Use proper hierarchy (don't skip levels)
- Make headings descriptive
- One H1 per page

## Multilingual Considerations (Future)

Currently English-only. If adding languages:

1. **Prioritize:** Hindi, then other Indian languages
2. **Localize:** Not just translate - adapt for culture
3. **Maintain:** Same tone and voice in all languages
4. **Technical:** Use i18n routing, not separate sites

## Content Approval Process

For major content changes:

1. Draft content in markdown
2. Review for tone, clarity, accuracy
3. Check SEO implications
4. Test on staging (if available)
5. Get feedback from team/users
6. Deploy to production
7. Monitor user feedback
